# Archimedies

A Game engine for people who are tired of closed standards. Uses open technologies, creates standards
where standards are not in place. 

## Goals

Godot is fantastic in taking the world of popular game engines by storm, the developer expirence in Godot goes two ways,
"wow this is fantastic", or "Wow this is horrible". And the same goes for any other popular tool in the current ecosystem of engines.
Unreal is heavy and has tools most people don't need or want. Unity is Unity. Bloat of these large editors will eventually make them need to undergo major changes.
We want to create a Open engine that not only gives you an editing expirence as expected, but also the ability to write your own tools / engines yourself.
Don't want the phsyiscs write your own. Dont want our renderer write your own.

a plugin system makes this possible. Pick and choose what you want and use your own APIS to write the rest.
Either you write nothing or something the editor is exactly what you make of it. Everyhting customizable, everything scriptable. everything yours.

## Testing

The thoery is that everything is verified not that everything is covered, custom hooks should be verfied as working.
At scale something like fuzz testing may be in order. Imagine fuzz testing for different rendering methods.
